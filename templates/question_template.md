# Question Template

このテンプレートを使用して新しい問題を作成してください。

```markdown
# Question XXX: [問題タイトル]

## Scenario
[シナリオ・状況説明]
- 組織の背景
- チームの状況
- 発生している問題や課題
- 制約条件

## Question
[問題文]
- 具体的に何を求められているか
- 回答の形式や長さの指示
- 考慮すべき観点

## Evaluation Criteria
1. **理解度** - Scrumの概念や原則の正確な理解
2. **適用力** - 実践的な状況での適用能力
3. **分析力** - 問題の分析と解決策の提案
4. **コミュニケーション** - 明確で説得力のある説明

## Model Answer
[模範解答]
- 期待される回答の内容
- 重要なポイント
- 回答の構造

## Key Points
- [重要ポイント1]
- [重要ポイント2]
- [重要ポイント3]
- [重要ポイント4]

## Difficulty Level
- Beginner
- Intermediate  
- Advanced

## Tags
- [タグ1]
- [タグ2]
- [タグ3]
```

## 使用方法

### 問題の作成
1. 上記テンプレートをコピーし、`questions/question_XXX.md`として保存（XXXは3桁の番号：001, 002, 003...）
2. 各セクションを具体的な内容で埋める（評価観点は問題の性質に応じて調整可能）

### 問題の出題（Agent向け）
問題を出題する際は、以下の手順を実行してください：

1. **使用済み問題の確認**: `sessions/`ディレクトリ内の`question.md`を検索し、既に使用された問題番号を特定
2. **問題の選択**: 
   - ユーザーが特定の問題番号を指定している場合は、その問題を選択
   - 指定がない場合は、未使用の問題からランダムに選択する
   - 複数問を出題する場合は、重複しないように注意する
3. **セッション作成**: 現在の日時（`YYYY-MM-DD_HH-MM`形式）でセッションディレクトリを作成
4. **ファイル作成**: セッションディレクトリ内に`question.md`（問題内容）、`answer.md`（回答用）、`feedback.md`（添削結果用）を作成

#### 複数問を出題する場合

複数問（例：2問）を出題する場合は、以下の形式に従ってください：

**question.md の形式**:
```markdown
# Question XXX: [問題タイトル]

## Scenario
[シナリオ]

## Question
[問題文]

---

# Question YYY: [問題タイトル]

## Scenario
[シナリオ]

## Question
[問題文]

```

**answer.md の形式**:
```markdown
# 回答

## Question XXX への回答

（ここに回答を記入してください）

## Question YYY への回答

（ここに回答を記入してください）
```

**注意事項**:
- 既存の問題ファイルから問題内容を正確にコピーしてください
- セッションディレクトリ名は必ず`YYYY-MM-DD_HH-MM`形式で作成してください
- 複数問を選択する場合は、異なる問題を選択してください
- `feedback.md`は初期状態は空でOKです
