# Question 010: スプリントレビューにおける経験主義の実現

## Scenario
スプリントレビューはスクラムの重要なイベントの一つです。このイベントは経験主義を実現するために重要な役割を果たしますが、すべての開発チームメンバーが参加できない場合、その効果に影響が出る可能性があります。

## Question
スプリントレビューはどのように経験主義を可能にしますか？開発チームの一部のメンバーが不在の場合、どのような影響がありますか？

## Evaluation Criteria
1. **スクラムガイドの理解** - スプリントレビューの目的と経験主義の3つの柱（透明性、検査、適応）との関係の理解
2. **実践的な適用** - 経験主義を実現するための具体的なメカニズムの説明
3. **影響分析** - 開発チームメンバーの不在がもたらす具体的な影響の分析
4. **学習目標達成** - 両方のサブクエスチョンに対する完全な回答と明確性

## Model Answer

### スプリントレビューが経験主義を可能にする方法

スプリントレビューは、スクラムチームと利害関係者がインクリメントを検査し、プロダクトバックログを適応させる機会を提供することで、経験主義を可能にします。このイベントは以下の方法で経験主義を実現します：

**透明性の促進**
- スプリントレビューは透明性をサポートし、全員が作成された成果物を確認できます
- ステークホルダーからの良質なフィードバックを収集することで、インクリメントの品質の透明性を高めます
- 実際の成果物を提示することで、進捗の現状を明確にします

**検査の機会**
- スクラムチームとステークホルダーが一緒にインクリメントを検査します
- スプリントゴールに向けた進捗を共同で評価する機会を提供します
- 実装された機能とその動作を実際に確認できます

**適応の機会**
- プロダクトバックログを適応させ、次のステップを決定します
- フィードバックに基づいて製品の方向性を調整できます
- 実証的なデータ（実際の成果物と反応）に基づいて判断を行います

### 開発チームの一部のメンバーが不在の場合の影響

開発チームの一部のメンバーが不在の場合、以下のような重大な影響があります：

**透明性の低下**
- 不在の開発チームメンバーは、レビュー中に提供されたフィードバックを直接聞く機会を失います
- ステークホルダーからの直接のフィードバックを逃し、彼らのニーズや要望を深く理解する機会を失います
- 実際の成果物に対する直接的な反応を体験できません

**検査の質の低下**
- スプリントゴールに向けた進捗を共同で評価する機会を逃します
- 技術的な実装の詳細や課題を説明できないため、検査が表面的になる可能性があります
- インクリメントに対する包括的な視点が欠けます

**コラボレーションの減少**
- 内部コラボレーションのフィードバックループが減少します
- チーム内での知識共有と学習の機会が失われます
- 不在のメンバーと参加メンバーの間で情報の非対称性が生じます

**適応の遅延と追加作業**
- 問題の発見や障害の対処、スプリントバックログの適応が遅れる可能性があります
- ステークホルダーへの情報提供が遅れる可能性があります
- 不在のメンバーに情報を伝えるための追加の会議が必要となり、作業の流れが妨げられる可能性があります

**チームの効率と製品の品質への影響**
- これらの要素は、チームの効率と製品の品質に悪影響を及ぼす可能性があります
- 次回のスプリントプランニングで不完全な情報に基づいて判断するリスクがあります

## Key Points
- スプリントレビューは透明性、検査、適応の3つの柱を通じて経験主義を実現
- インクリメントの検査とプロダクトバックログの適応の機会を提供
- ステークホルダーからの直接フィードバックが重要
- 開発チームメンバーの不在は透明性の低下を引き起こす
- 直接のフィードバックを聞く機会の損失
- 共同評価とコラボレーションの機会の損失
- 追加の会議が必要となり、作業の流れが妨げられる可能性
- チームの効率と製品の品質への悪影響
